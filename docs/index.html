Nothing Health API

Access health and fitness data from Nothing watches. Build dashboards, AI coaching tools, research integrations, and more — with user consent and scoped permissions.

Base URL

https://api.nothing.tech/v1

What you can build

The Nothing Health API provides read access to health metrics collected by Nothing watches. Common use cases include:

  • Health dashboards and data visualization
  • AI-powered fitness coaching and insights
  • Integration with health platforms and EHR systems
  • Research and wellness applications
  • MCP servers for AI assistant integrations

Quickstart

Get up and running in three steps.

1. Register your application

Create an app in the Developer Portal to get your client credentials.

POST /api/v1/oauth/register

{
  "name": "My Health App",
  "redirect_uri": "https://myapp.com/callback",
  "scopes": "heart_rate:read sleep:read steps:read"
}

2. Get user authorization

Redirect users to the Nothing consent screen. They choose exactly which data to share.

POST /api/v1/oauth/authorize

{
  "client_id": "your_client_id",
  "redirect_uri": "https://myapp.com/callback",
  "scope": "heart_rate:read sleep:read",
  "response_type": "code"
}

3. Exchange code for tokens

POST /api/v1/oauth/token

{
  "grant_type": "authorization_code",
  "code": "the_auth_code",
  "client_id": "your_client_id",
  "client_secret": "your_client_secret",
  "redirect_uri": "https://myapp.com/callback"
}

You'll receive an access token and refresh token. Use the access token in the Authorization header:

GET /api/v1/health/heart_rate
Authorization: Bearer your_access_token

Authentication

The Nothing Health API uses OAuth 2.0 with authorization codes. Users explicitly consent to share specific data types with your application.

Token lifecycle

TokenLifetimeUsage
access_token 30 minutes Passed in Authorization: Bearer header
refresh_token 90 days Exchange for a new access token via /oauth/token
authorization_code 5 minutes One-time use, exchanged for tokens

Revoking access

Users can revoke your app's access at any time from the Nothing X app with a single tap. You can also programmatically revoke tokens: 

POST /api/v1/oauth/revoke

{
  "token": "the_refresh_token",
  "client_id": "your_client_id",
  "client_secret": "your_client_secret"
}

Heart Rate

Continuous and on-demand heart rate measurements from the optical sensor.

GET /api/v1/health/heart_rate

Returns heart rate readings for the authenticated user.

Query parameters

ParameterTypeDescription
startISO 8601Filter readings after this time
endISO 8601Filter readings before this time
limitintMax results (default 50, max 200)
offsetintPagination offset

Response

{
  "data": [
    {
      "id": "abc-123",
      "timestamp": "2026-03-27T08:15:00Z",
      "bpm": 72,
      "source": "watch"
    }
  ],
  "has_more": true,
  "staleness": {
    "last_synced_at": "2026-03-27T08:00:00Z",
    "sync_interval_minutes": 30
  }
}

Requires scope: heart_rate:read

Scopes & Permissions

Users grant access to specific data types when authorizing your app. Request only the scopes your application needs.

heart_rate:read
Heart rate measurements
sleep:read
Sleep sessions and stages
spo2:read
Blood oxygen readings
steps:read
Daily step counts
workouts:read
Exercise sessions
skin_temperature:read
Skin temperature readings
menstrual_cycle:read
Menstrual cycle tracking
profile:read
Basic user profile
Principle of least privilege

Only request the scopes your app actually uses. Apps requesting unnecessary scopes may be rejected during review.

Rate Limits

Rate limits are applied per application (identified by client ID).

WindowLimit
Per minute100 requests
Per day10,000 requests

Rate limit status is returned in response headers:

X-RateLimit-Limit-Minute: 100
X-RateLimit-Remaining-Minute: 95
X-RateLimit-Limit-Day: 10000

When rate limited, the API returns 429 Too Many Requests with a Retry-After header.

Data Freshness

Health data syncs from the watch to Nothing servers via the Nothing X app in the background. The current sync interval is approximately 30 minutes.

Design around sync cadence

Every health data response includes a staleness object with last_synced_at and sync_interval_minutes. Use this to show users when their data was last updated.

Data may be delayed if the user's phone is off, the Nothing X app is force-closed, or the watch is out of Bluetooth range. Your app should handle stale data gracefully.

App Review

All applications must pass review before accessing production data. During development, you can use sandbox credentials with test data.

Review criteria

  • Clear description of how health data will be used
  • Privacy policy that covers health data handling
  • Only requests scopes that the app actually uses
  • HTTPS redirect URIs only
  • Does not resell or share raw health data with third parties
Timeline

App reviews are typically completed within 5 business days.
